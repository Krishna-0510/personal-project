Krishna Kirana Stores — Step-by-Step Implementation Roadmap
Here's your complete implementation plan broken into clear phases. Each step is small, focused, and testable before moving to the next.

🗂️ PHASE 0 — Setup & Accounts (Before Writing Any Code)
Step 1 — Create all accounts first

GitHub account → create a private repository named krishna-kirana
MongoDB Atlas → create free cluster → create database user → whitelist your IP
Firebase → create new project → enable Phone Auth + Google Auth → get config keys
Twilio → create account → get a WhatsApp sandbox number → note Account SID and Auth Token
Vercel → connect your GitHub account
Render → connect your GitHub account
Cloudflare → add your domain (free plan)

Step 2 — Prepare your local machine

Install Node.js (v18+), Git, VS Code
Install VS Code extensions: ESLint, Prettier, Tailwind CSS IntelliSense, MongoDB
Install globally: nodemon, vite
Create the full folder structure exactly as shown in the prompt before writing any code

Step 3 — Prepare your .env files

Create client/.env and server/.env
Fill in every variable from the prompt with your real keys
Add both .env files to .gitignore immediately
Keep a private offline copy of all keys (notepad/phone)


🔧 PHASE 1 — Backend Foundation
Step 4 — Basic Express server

Create server.js with Express
Add helmet.js, cors, dotenv, express.json()
Set CORS to allow only your frontend URL
Add a test route GET /health that returns { status: "ok" }
Run server, confirm it works on localhost:5000

Step 5 — MongoDB connection

Connect Mongoose to your Atlas URI
Test that connection log shows "MongoDB connected"
Create all 6 models one by one: User, Product, Order, Combo, Coupon, Recharge
Each model should have all fields from your feature list

Step 6 — Middleware setup

Create verifyToken.js — checks JWT from httpOnly cookie
Create isAdmin.js — checks if user has admin role
Create rateLimiter.js — max 5 orders/user/hour, 20/IP/day
Test each middleware with a simple dummy route

Step 7 — Firebase Admin setup

Install Firebase Admin SDK in server
Initialize with your service account credentials from .env
Create a function verifyFirebaseToken(idToken) that returns the user's UID and phone number
Test it with a dummy token


🔐 PHASE 2 — Authentication
Step 8 — Auth routes (backend)

POST /api/auth/verify-otp → receives Firebase ID token → verifies with Admin SDK → creates/finds user in MongoDB → issues JWT → sets httpOnly cookie
POST /api/auth/logout → clears cookie
GET /api/auth/me → returns current user (requires verifyToken middleware)

Step 9 — Firebase frontend setup

Add Firebase config to client/src/firebase/config.js
Enable Phone Auth with reCAPTCHA in Firebase console
Enable Google Sign-In in Firebase console

Step 10 — AuthContext

Create AuthContext.jsx with: currentUser, login, logout, loading state
Wrap entire app in this context in App.jsx
Handle auto-login on page refresh using stored cookie

Step 11 — Login page

Phone number input → send OTP via Firebase
OTP input → verify → call backend /verify-otp
Google Sign-In button → Firebase popup → call same backend route
Guest browsing: skip login, just close modal
Protect checkout route: redirect to login if not authenticated


📦 PHASE 3 — Products & Categories
Step 12 — Product API (backend)

GET /api/products → all products with filters (category, search, sort)
GET /api/products/:id → single product
POST /api/products → admin only, add product
PUT /api/products/:id → admin only, update
DELETE /api/products/:id → admin only
Add Joi validation to all input routes

Step 13 — Seed your product database

Write a one-time seed script in server/utils/seedProducts.js
Add all products from the prompt (Rice & Grains, Pulses, Dairy, etc.)
Add all 3 language names per product (English, Gujarati, Hindi)
Run it once, verify products appear in MongoDB Atlas

Step 14 — Combo API (backend)

GET /api/combos → all active combos
POST /api/combos → admin only
PUT /api/combos/:id → admin only
Seed the 4 combos from the prompt with their savings amounts

Step 15 — Seasonal product logic

Add fields to Product model: isSeasonal, seasonStart, seasonEnd, seasonalPrice
Create a MongoDB query that auto-filters seasonal products by today's date
GET /api/products/seasonal → only products where today is within their season range
Seed seasonal products for all 5 seasons from the prompt


🛒 PHASE 4 — Orders & Cutoff System
Step 16 — Order API (backend)

POST /api/orders → place order → requires verifyToken
GET /api/orders/my → customer's own orders
GET /api/orders → admin only, all orders
PUT /api/orders/:id/status → admin only, update status
DELETE /api/orders/:id/cancel → customer cancel within 5 minutes

Step 17 — Order cutoff logic

Create server/utils/scheduler.js
Use node-cron to run job at 15 21 * * * (9:15 PM IST)
At cutoff: set a global flag in DB (e.g., a Settings document field ordersOpenForToday: false)
All new orders after cutoff automatically go to tomorrow's date
At midnight: reset flag back to true
Add a GET /api/settings/cutoff-status route that frontend polls

Step 18 — Pickup slot logic

Store slot rules in Settings: Morning 9–11 AM (order by 8 AM), Afternoon 2–4 PM (order by 1 PM), Evening 6–8 PM (order by 5 PM)
Create utility function getAvailableSlots(now) that returns which slots are still open
Expose as GET /api/orders/available-slots
After 9:15 PM, all slots return as tomorrow's slots


💬 PHASE 5 — WhatsApp Integration (Twilio)
Step 19 — Twilio setup

Install Twilio SDK in server
Create server/utils/whatsapp.js with a sendWhatsApp(to, message) function
Test by sending a WhatsApp message to your own number
Set up Twilio webhook URL pointing to POST /api/webhook/whatsapp

Step 20 — Order notification to father

After order is placed in Step 16, call sendWhatsApp with the formatted order message from the prompt
Include all order details and the reply menu (1–5)

Step 21 — Father's reply webhook

Create POST /api/webhook/whatsapp route
Parse incoming message: check if sender is father's number
Match reply number (1–5) to action
Update order status in MongoDB
Send appropriate customer notification via WhatsApp
Test each reply number one by one

Step 22 — Price update via WhatsApp

In the same webhook, detect if message starts with "price" or "prices" or "stock"
Parse the command format: price [name] [qty] [amount]
Use fuzzy matching (install fuse.js) to find product by name even with spelling mistakes
Update product price or stock in MongoDB
Log change in a PriceChangeLogs collection: timestamp, old price, new price, product
Send confirmation back to father
Test with: "price rice 5kg 280"

Step 23 — Daily cutoff WhatsApp to father

In the scheduler from Step 17, after closing orders, calculate today's totals
Send WhatsApp to father with order count and revenue summary
Test by temporarily changing cron time to 1 minute from now

Step 24 — Recharge WhatsApp flow

POST /api/recharge/request → saves request to DB, sends WhatsApp to father
In webhook: detect reply 1 or 2 for recharge (distinguish from order replies using message context)
Send customer success or failure WhatsApp
Test full cycle: submit → father gets message → reply → customer gets message

DATABASE SCHEMA :

1. USER
User
├── _id                     ObjectId (auto)
├── firebaseUid             String, required, unique
├── name                    String
├── mobile                  String, required, unique (+91XXXXXXXXXX)
├── email                   String, optional
├── role                    String, enum: ["customer", "admin"], default: "customer"
├── isBlocked               Boolean, default: false
├── fcmToken                String (for push notifications)
├── preferredLanguage       String, enum: ["en", "gu", "hi"], default: "en"
│
├── loyaltyPoints           Number, default: 0
├── totalSpent              Number, default: 0
├── referralCode            String, unique (auto-generated on register)
├── referredBy              String (referral code of who referred them)
├── referralCount           Number, default: 0
│
├── firstOrderDone          Boolean, default: false
├── lastOrderDate           Date
├── inactiveReminderSent    Boolean, default: false
│
├── savedAddresses []
│   ├── label               String (Home / Work / Other)
│   ├── line1               String
│   ├── landmark            String
│   ├── pincode             String
│   └── isDefault           Boolean
│
└── timestamps              createdAt, updatedAt

2. PRODUCT
Product
├── _id                     ObjectId (auto)
├── sku                     String, unique (e.g. "RICE-BASMATI-5KG")
│
├── name
│   ├── en                  String, required
│   ├── gu                  String
│   └── hi                  String
│
├── description
│   ├── en                  String
│   ├── gu                  String
│   └── hi                  String
│
├── category                String, enum:
│                           ["rice_grains", "pulses", "dairy",
│                            "bread_eggs", "daily_staples",
│                            "spices", "seasonal", "combo"]
│
├── images []               Array of Strings (image URLs)
├── primaryImage            String (main display image URL)
│
├── price                   Number, required (our selling price ₹)
├── marketPrice             Number (MRP / market price for comparison badge)
├── unit                    String (e.g. "5kg", "1L", "500g", "dozen")
│
├── stockQty                Number, default: 0
├── stockAlertLevel         Number, default: 10
├── inStock                 Boolean, default: true
│
├── isSeasonal              Boolean, default: false
├── seasonName              String, enum:
│                           ["monsoon", "diwali", "winter",
│                            "navratri", "summer", null]
├── seasonStart             Date
├── seasonEnd               Date
├── seasonalPrice           Number (special price during season)
│
├── isActive                Boolean, default: true
├── isFeatured              Boolean, default: false
├── tags []                 Array of Strings (for search)
│
├── salesCount              Number, default: 0 (for "popular" sort)
│
└── timestamps              createdAt, updatedAt

3. ORDER
Order
├── _id                     ObjectId (auto)
├── orderNumber             String, unique (e.g. "KK-2024-0001")
├── customer                ObjectId → ref: User, required
│
├── items []
│   ├── product             ObjectId → ref: Product
│   ├── productName         String (snapshot at order time)
│   ├── productImage        String (snapshot)
│   ├── qty                 Number
│   ├── unitPrice           Number (price at time of order)
│   └── subtotal            Number
│
├── comboApplied            ObjectId → ref: Combo (if combo used)
├── couponApplied           ObjectId → ref: Coupon (if coupon used)
├── discountAmount          Number, default: 0
├── loyaltyPointsUsed       Number, default: 0
├── loyaltyDiscountAmount   Number, default: 0
│
├── subtotal                Number
├── totalAmount             Number (after all discounts)
│
├── paymentMethod           String, enum: ["upi", "pay_at_shop"]
├── paymentStatus           String, enum:
│                           ["pending", "paid", "failed"],
│                           default: "pending"
│
├── pickupSlot
│   ├── date                Date
│   ├── slot                String, enum:
│   │                       ["morning_9_11", "afternoon_2_4", "evening_6_8"]
│   └── label               String (e.g. "Morning 9–11 AM")
│
├── status                  String, enum:
│                           ["placed", "accepted", "ready",
│                            "collected", "cancelled", "declined"],
│                           default: "placed"
│
├── statusHistory []
│   ├── status              String
│   ├── changedAt           Date
│   └── changedBy           String ("father_whatsapp" / "admin_panel")
│
├── isForTomorrow           Boolean, default: false
│   (true if placed after 9:15 PM cutoff)
│
├── cancelledAt             Date
├── cancelReason            String
├── canCancelUntil          Date (5 minutes after placement)
│
├── whatsappSentToFather    Boolean, default: false
├── whatsappSentAt          Date
│
├── rating                  Number (1–5, filled after collected)
├── review                  String
│
└── timestamps              createdAt, updatedAt

4. COMBO
Combo
├── _id                     ObjectId (auto)
├── name
│   ├── en                  String, required
│   ├── gu                  String
│   └── hi                  String
│
├── description
│   ├── en                  String
│   ├── gu                  String
│   └── hi                  String
│
├── image                   String (combo banner image URL)
│
├── products []
│   ├── product             ObjectId → ref: Product
│   ├── qty                 Number
│   └── unitPrice           Number (price snapshot)
│
├── originalTotal           Number (sum of all items at regular price)
├── comboPrice              Number (discounted combo price)
├── savingsAmount           Number (originalTotal - comboPrice)
├── savingsPercent          Number
│
├── isSeasonal              Boolean, default: false
├── seasonName              String
├── validFrom               Date
├── validUntil              Date
│
├── isActive                Boolean, default: true
├── displayOrder            Number (for sorting on UI)
│
└── timestamps              createdAt, updatedAt

5. COUPON
Coupon
├── _id                     ObjectId (auto)
├── code                    String, required, unique, uppercase
│   (e.g. "FIRST10", "KRISHNA20")
│
├── description             String
├── discountType            String, enum: ["percent", "flat"]
├── discountValue           Number
│   (10 means 10% if percent, ₹10 if flat)
│
├── minOrderAmount          Number, default: 0
├── maxDiscountCap          Number (max ₹ discount even if percent)
│
├── usageLimitTotal         Number (max total uses, null = unlimited)
├── usageLimitPerUser       Number, default: 1
├── usedCount               Number, default: 0
│
├── usedBy []               Array of ObjectId → ref: User
│
├── isActive                Boolean, default: true
├── validFrom               Date
├── validUntil              Date
│
├── applicableFor           String, enum:
│                           ["all", "first_order", "referral",
│                            "festival", "specific_category"]
├── applicableCategory      String (if applicableFor = specific_category)
│
└── timestamps              createdAt, updatedAt

6. RECHARGE
Recharge
├── _id                     ObjectId (auto)
├── requestNumber           String, unique (e.g. "RC-2024-0001")
│
├── customer                ObjectId → ref: User
├── customerName            String
├── customerMobile          String (submitter's number)
│
├── operator                String, enum: ["jio", "vi", "airtel"]
├── rechargeNumber          String (number to recharge, 10 digits)
├── amount                  Number
│
├── paymentMethod           String, enum: ["pay_at_shop", "upi"]
│
├── status                  String, enum:
│                           ["pending", "completed", "failed"],
│                           default: "pending"
│
├── whatsappSentToFather    Boolean, default: false
├── fatherRepliedAt         Date
├── completedAt             Date
│
├── customerNotified        Boolean, default: false
│
└── timestamps              createdAt, updatedAt

7. PRICE CHANGE LOG
PriceChangeLog
├── _id                     ObjectId (auto)
├── product                 ObjectId → ref: Product
├── productName             String (snapshot)
├── productSku              String (snapshot)
│
├── oldPrice                Number
├── newPrice                Number
├── changeAmount            Number (newPrice - oldPrice)
│
├── changedVia              String, enum:
│                           ["whatsapp_command", "admin_panel",
│                            "bulk_update"]
│
├── changedBy               String
│   ("father_whatsapp" / "admin" / "system")
│
├── rawCommand              String
│   (original WhatsApp text if via whatsapp_command,
│    e.g. "price rice 5kg 280")
│
└── createdAt               Date (timestamp of change)
